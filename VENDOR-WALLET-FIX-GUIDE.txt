📋 PROBLEM SUMMARY
==================
Orders placed with wallet payment did not credit vendor wallets because:
- Products/Stores use vendorId as string identifiers (e.g., "JLC")
- User accounts use MongoDB ObjectId for _id
- Stores were not linked to user accounts via linkedWalletUserId

✅ WHAT WAS FIXED
==================
1. ✓ Improved creditVendorWalletsForOrder to safely handle vendor ID resolution
2. ✓ Added storeId field to Product model for better linking
3. ✓ Auto-link stores to vendor user accounts during creation
4. ✓ Auto-populate product storeId during creation
5. ✓ Created utility scripts for fixing existing data

🔧 IMMEDIATE ACTION REQUIRED
==============================

STEP 1: Link Existing Stores to User Accounts
----------------------------------------------
Run this command to automatically link stores to vendor users:

  node link-stores-to-users.js

This matches stores with vendors by business name. If any stores can't
be auto-matched, you'll see a list of vendors to manually link.

For manual linking:
  node manual-store-link.js <vendorId> <userId>

Example:
  node manual-store-link.js JLC 6992d29fe8074a5db2b5e588


STEP 2: Backfill Product Store IDs (Recommended)
-------------------------------------------------
Run this to link all products to their stores:

  node backfill-product-storeids.js


STEP 3: Credit Vendors for Failed Orders
-----------------------------------------
First, list recent orders to find the order ID:

  node list-recent-orders.js

Then retry crediting for specific orders:

  node retry-vendor-credit.js <orderId>

Example output:
  --- Processing Vendor: JLC ---
  Amount: ₦321
  Store: JLC Store (64abc123...)
  ✓ Credited vendor wallet: YES
  ✓ Credited store wallet: YES


🔍 VERIFICATION

📊 TECHNICAL DETAILS
=====================

Data Structure:
  Product.vendorId (string) → "JLC"
  Store.vendorId (string) → "JLC"
  Store.linkedWalletUserId (ObjectId) → User._id
  User._id (ObjectId) → "6992d29fe8074a5db2b5e588"

Payment Flow (Fixed):
1. Customer pays with wallet
2. Order created with vendor items
3. creditVendorWalletsForOrder runs:
   - Looks up Store by vendorId (string)
   - Gets linkedWalletUserId from Store
   - Credits User wallet using ObjectId
   - Credits Store wallet
   - Creates WalletTransaction record

🚨 TROUBLESHOOTING
===================

Error: "Store not linked to user wallet"
→ Solution: Run manual-store-link.js for that vendor

Error: "User not found"  
→ The vendor may not have a user account
→ Check: node check-vendor-wallets.js
→ Ensure vendor signed up and has account

Error: "Already credited"
→ Transaction already exists - vendor was paid
→ Check wallet balance to confirm
